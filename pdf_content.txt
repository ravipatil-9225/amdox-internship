1 Executive Overview  
 
Mission statement  
NeuralRetail is an end -to-end AI -powered sales intelligence platform designed for Amdox's enterprise clients in retail, 
FMCG, and e -commerce. The platform ingests multi -source transactional, behavioural, and external data to produce 
accurate demand forecas ts, actionable customer intelligence, churn predictions, and inventory optimization 
recommendations — all served through an interactive analytics dashboard and REST API.  
 
Why NeuralRetail matters for Amdox Technologies  
Retailers globally lose an estimated $1.75 trillion annually from stockouts and overstocking. NeuralRetail positions 
Amdox as a full -stack data intelligence partner, delivering 30 –50% stockout reduction, 15 –25% revenue uplift, and 20 –
35% improvement in cus tomer retention. The platform's MLOps backbone ensures models stay accurate over time 
without manual intervention — a genuine competitive differentiator.

5 Granular 28 -Day Execution Plan  
 
Week 1 – Data discovery, ingestion & feature engineering  
Day 1 – Project setup & data discovery  
• Stakeholder requirements workshop: define KPI targets, data sources, and success criteria  
• Competitor analysis: Databricks Retail AI, Google Cloud Retail, AWS Forecast, Azure Synapse  
• Dataset acquisition: Kaggle RetailRocket, M5 Forecasting Competition, UCI Online Retail II  
• Monorepo setup: Python + Poetry + Ruff + Black + pre -commit hooks + conventional commits  
• Docker Compose local stack: PostgreSQL 17, Redis 8, MLflow, Airflow, Streamlit  
Day 2 – Data ingestion pipeline  
• Spark ingestion job: read CSV/Parquet sources → Delta Lake bronze layer with schema enforcement  
• Great Expectations suite: define expectations on row counts, nullability, value ranges, foreign keys  
• OpenLineage metadata emission to local Marquez instance for lineage tracking  
• Airflow DAG skeleton: ingest_bronze → validate_dq → notify_on_failure  
• Unit tests for ingestion functions; mock S3 with moto for CI compatibility  
Day 3 – EDA & feature engineering  
• EDA notebook: distribution analysis, outlier detection, correlation heatmap, seasonality decomposition  
• Feature engineering: RFM scores, rolling mean/std demand (7/14/30 days), lag features (t -1, t-7, t-14) 
• Date features: day -of-week, week -of-year, holiday flags, promotional event flags  
• External data join: weather (Open -Meteo API), macro CPI index by product category  
• Silver layer Delta table: clean, enriched, feature -ready dataset with data versioning  
Day 4 – Feature store & data quality  
• Feast feature definitions: customer, product, and transaction feature views  
• Online store materialization to Redis for low -latency serving at inference time  
• Offline store to S3 Parquet for training dataset retrieval with point -in-time joins  
• Great Expectations checkpoint integration into Airflow with Slack alerting on DQ failures  
• Data profiling report: YData Profiling HTML export for stakeholder review  
Day 5 – Baseline models & MLflow setup  
• MLflow tracking server setup: PostgreSQL backend + S3 artifact store + experiment UI  
• Baseline demand model: Prophet with default seasonality; log MAPE, RMSE to MLflow  
• Baseline churn model: Logistic Regression; log AUC -ROC, F1, confusion matrix to MLflow  
• MLflow Model Registry: register first baseline models as 'Staging' for version tracking  
• Experiment comparison dashboard in MLflow UI; document baseline as benchmark  
Day 6 – Time -series deep dive  
• Stationarity tests: ADF, KPSS; differencing and log -transform where needed  
• STL decomposition: trend, seasonality, residual analysis for top -20 revenue SKUs  
• Autocorrelation and PACF analysis to inform ARIMA order selection  
• Multi -seasonality detection: weekly + annual patterns; Prophet with dual seasonality  
• Hyperparameter grid search for Prophet: changepoint_prior_scale, seasonality_mode  
Day 7 – Week 1 checkpoint  
• Deliverables: EDA HTML report, cleaned Delta Lake silver table, baseline models in MLflow

Week 2 – Advanced ML models & ensemble construction  
Day 8 – LSTM forecasting model  
• PyTorch Lightning LSTM: window -based input (28 -day lookback), multi -step output (1/7/30 day)  
• TorchForecast TFT (Temporal Fusion Transformer) as secondary deep learning architecture  
• Training loop: gradient clipping, learning rate scheduler (CosineAnnealingWarmRestarts)  
• Probabilistic output: quantile regression (q10, q50, q90) for safety -stock interval estimation  
• MLflow autolog: hyperparameters, training curves, validation MAPE per forecast horizon  
Day 9 – Prophet + LSTM ensemble  
• Ensemble strategy: weighted average with weights optimized by Optuna on validation set  
• NeuralProphet as intermediate model bridging statistical and deep learning approaches  
• TimeGPT zero -shot forecast as benchmark comparison (API call, no fine -tuning required)  
• Final ensemble MAPE target: ≤ 10% on 30 -day horizon across all ABC -classified SKU tiers  
• Confidence interval calibration: isotonic regression on quantile outputs for reliability  
Day 10 – Churn prediction model  
• Feature set: RFM metrics, purchase frequency decay, category switching signals, recency cliff  
• XGBoost: Bayesian hyperparameter optimization with Optuna (100 trials, 5 -fold stratified CV)  
• LightGBM: DART boosting mode for regularization; benchmark vs XGBoost on AUC -ROC 
• Stacking: meta -learner (Logistic Regression) on XGB + LGBM + RF out -of-fold predictions  
• Class imbalance: SMOTE -NC oversampling + scale_pos_weight; evaluate at threshold=0.35  
Day 11 – SHAP explainability & model interpretation  
• SHAP TreeExplainer for XGBoost churn model: global feature importance + local explanations  
• SHAP beeswarm, waterfall, and dependence plots for top -10 predictive features  
• LIME explanations as secondary interpretability check for model robustness validation  
• Captum integrated gradients for LSTM temporal feature attribution visualization  
• Business narrative: translate SHAP values into plain -language retention action recommendations  
Day 12 – Customer segmentation & RFM intelligence  
• K-Means: Elbow + Silhouette analysis (k=6 –10); final k selected by Davies -Bouldin index  
• DBSCAN: auto eps via k -distance graph; identify outlier / ultra -VIP customer clusters  
• Gaussian Mixture Models: probabilistic segment membership for borderline customers  
• Segment profiling: revenue contribution, churn risk, category affinity, CLV estimate per cluster  
• CRM export: high -churn -risk segment with SHAP -based retention action codes  
Day 13 – Price elasticity & revenue intelligence  
• Log-log OLS regression: price vs demand per category with promotion dummy variables  
• DoWhy causal graph: identify and adjust for confounders (season, competitor price, ad spend)  
• EconML Double ML for heterogeneous treatment effects of price changes across segments  
• Promotion lift analysis: pre/post attribution with synthetic control method  
• What -if revenue simulator: adjust price/promotion inputs → see projected demand and revenue impact  
Day 14 – Week 2 checkpoint  
• Deliverables: ensemble forecast (MAPE ≤ 10%), churn model (AUC ≥ 0.90), segmentation, elasticity model  
• MLflow Model Registry: promote best models from Staging → Production after validation gate  
• Champion/challenger: new model must beat current champion by ≥ 5% on unseen hold -out set  
• Weekly model report: MAPE by SKU tier, AUC by cohort, segment stability — logged to MLflow
